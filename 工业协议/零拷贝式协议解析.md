- 零拷贝式的协议解析方式即 强转内存块为结构体

- 常见语言协议解析
  - elixir: <<>>
  - C: C 的 bit-fields
  - zig: packed struct
  - Nim: 零拷贝方式强转内存块为结构体
  - Rust: 零拷贝方式强转内存块为结构体, Rust 中，很少有人手写 unsafe 指针转换。大家通常使用 zerocopy 或 bytemuck
  - java 17+ 引入了 Foreign Function & Memory API; 它允许你定义 MemoryLayout（内存布局），然后通过 VarHandle 直接读写内存。这已经非常接近零拷贝解析，但语法上仍然是“方法调用”，而不是“属性访问”
