对于 Fresher（初学者），直接啃大部头的数学书会非常痛苦且容易劝退。最好的学习路径是**“自底向上”的动手实践**——先学会怎么“骗”过眼睛，再从数学上证明它。

以下是为你精选的学习路线和资料推荐，按从易到难排序：

第一阶段：直觉建立与 2D 模拟 3D (最推荐起步)
目标：不使用任何复杂的图形 API (OpenGL/WebGL)，仅用简单的画点画线 API（如 Canvas 2D），徒手写出一个旋转立方体。弄懂 sin/cos 和透视除法。

[必看] Coding Math (YouTube 频道)
作者: Keith Peters (早期的 Flash 动画大神)。
内容: 这是一个极其宝藏的系列。他手把手教你在 Canvas 上写代码。
重点集数:
Ep 1-10: 基础三角学 (Trigonometry)，解释 sin/cos 怎么让东西动起来。
Ep 24-29: 3D in 2D。这一段就是教你我是刚才写的那段代码的原理——如何手动计算透视、旋转、Z轴排序。
特点: 讲得极好，完全面向程序员，而不是数学家。
《HTML5 Canvas 核心技术》 (Core HTML5 Canvas)
也就是 Keith Peters 写的书（原名 Foundation HTML5 Animation with JavaScript 的升级版）。它是图形学入门的神书，它不讲 OpenGL，只讲物理、数学、动画原理。
第二阶段：真正的图形学管线 (WebGL/OpenGL)
目标：理解显卡是怎么工作的，什么是矩阵 (Matrix)，什么是着色器 (Shader)。

Games101 - 现代计算机图形学入门
讲师: 闫令琪 (加州大學聖塔芭芭拉分校)。
语言: 中文授课！
地位: 目前华语圈公认最好的图形学入门课。
内容: 从线性代数开始，讲到光栅化、光线追踪。非常硬核，但闫老师讲得深入浅出。如果你能坚持看完前 10 节课，你就已经超越了 90% 的前端/后端开发者的图形学水平。
WebGL Fundamentals
类型: 在线互动教程。
特点: 它不教你用 Three.js，而是教你写原生的 WebGL。很多人觉得 WebGL 难是因为不懂矩阵，这个网站用无数个可交互的图表（也就是我们之前说的 Interactive Book）要把矩阵乘法给你讲懂。
第三阶段：数学内功 (线性代数)
目标：当你发现旋转矩阵推导看不懂，或者想做更复杂的物理模拟时。

3Blue1Brown - 线性代数的本质 (Essence of linear algebra)
类型: B站/YouTube 视频系列。
评价: 神作。它不会让你去背行列式计算公式，而是通过可视化动画让你看到矩阵到底是如何“拉伸”、“旋转”空间的。看完这个，你会因为发现数学之美而起鸡皮疙瘩。
总结建议
不要一开始就去学 Three.js。 Three.js 封装得太好了，你会 box.rotation.y += 0.01，但你依然不知道为什么它会转。

请按这个顺序：

看 Coding Math (或类似教程)，用 Canvas 2D 手写一个 3D 立方体。
看 3Blue1Brown，顿悟矩阵的几何意义。
看 Games101，系统学习图形学。
