### 名义类型系统（Nominal Typing） 是计算机科学中类型检查的一种机制。它的核心逻辑非常简单直接：
- 两个变量是否属于同一种类型，完全取决于它们“显式声明的名字”
- 即：即便两个类型的内部结构（成员变量、方法）一模一样，只要它们声明的名字不同，编译器就认为它们是不同的
- Java 是典型的名义类型语言
  ```java
  class UserID {
    public int value;
  }
  
  class PostID {
      public int value;
  }
  
  // 在 Java 中：
  UserID uid = new UserID();
  PostID pid = new PostID();
  
  // uid = pid; // ❌ 编译报错！
  // 尽管 UserID 和 PostID 内部都只有一个 int value，
  // 但在 Java 看来，UserID 就是 UserID，PostID 就是 PostID。
  ```

### 类比  
- 名义类型的现实比喻
  - 名义类型（Nominal）：就像身份证。
  - 双胞胎兄弟 A 和 B 即使长得一模一样（结构相同），但身份证号不同，在法律（编译器）眼里他们就是两个独立的人，不能互相替代。
- 结构化类型（Structural）：就像人脸识别。
  - 只要你的脸部特征（属性、方法）符合定义的特征库，系统就认为你是“合法的访问者”，不管你叫什么名字
